使用Hexo+Github一步步搭建属于自己的博客

Posted on March 26, 2019 本文总阅读量次

1 名词介绍

1.1 Hexo

Hexo 是一款基于Node.js、快速、简洁且高效的博客框架。Hexo 使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。
Hexo官方文档
Hexo资料

1.2 Github Pages

Github Pages可以被认为是用户编写的、托管在github上的静态网页，由于他的空间免费稳定，用来做搭建一个博客再好不过了。
Github Pages官网

1.3 Markdown

Markdown是一种可以使用普通文本编辑器编写的标记语言，通过简单的标记语法，它可以使普通文本内容具有一定的格式。
Markdown快速入门

Tips：以下操作适用于Centos 7 x64， 如果你使用其他系统（Widows/Ubuntu/Macos）需参考当前所使用系统的软件安装及环境变量配置

2 环境配置

2.1 Git 的安装及配置

以下为 Cenost 7 系统的 Git 的安装方式，如果你使用其他系统（Widows/Ubuntu/Macos）需参考当前所使用系统的 Git 安装及环境变量配置

1. 安装 Git

yum install git

2. 查看 Git 版本

git --version

2.2 Node.js 的安装及配置

以下为 Cenost 7 系统的 Node.js 的安装方式，如果你使用其他系统（Widows/Ubuntu/Macos）需参考当前所使用系统的 Node.js 安装及环境变量配置

1. 下载 Node.js

curl --silent --location https://rpm.nodesource.com/setup_8.x | bash -

2. 安装 node.js

yum install -y nodejs

3. 查看 Node.js 版本

node -v

3 Hexo 的安装及配置

1. 安装Hexo

npm install -g hexo

2. 在自己认为合适的地方创个文件夹，进入该文件夹

mkdir blog && cd blog
blog文件夹便是你的博客目录，文件夹可随意命名，此时最好将此目录备份到云盘或者其他地方，以防文件夹丢失后博客就没有了。

3. 初始化目录

hexo init

4. 安装依赖

npm install
至此，博客搭建成功！当然，仅仅是本地的了。

5. 生成静态页面

hexo g

6. 生成预览

hexo s
访问 http:localhost:4000 即可看到你的博客。
问题：假如页面一直无法跳转，那么可能端口被占用了。此时我们ctrl+c停止，接着输入hexo server -p 端口号来改变端口号

4 Github账号的配置

这一步是Github创建仓库来创建GitHub Pages页面
Github账号注册 略…

4.1 创建仓库

  1. 点击头像->Your repositories(您的仓库)->点击[New]新建仓库
  2. 在Repository name（存储库)中填写[你的Github用户名.github.ok]
  3. 选择Public 公开此仓库，请勿选择Private私有化仓库，否则会导致无法创建GitHub Pages
  4. 勾选Initialize this repository with a README（使用自述文件初始化此存储库）

4.2 创建 GitHub Pages

  1. 选择刚刚创建好的仓库，点击[settings]设置仓库
  2. 找到 GitHub Pages 点击[Automatic Page Generator]选择主题并确认
  3. 访问 你的Github.github.io 查看你的博客

5 Hexo 连接 GitHub

1. 在本机博客目录下编辑 [config.yml] 文件

vi __confit.yml

2. 安装 hexo-deployer-git

npm install hexo-deployer-git --save

3. 将本地 blog 部署到 Github 的仓库

hexo clean
hexo g && hexo d

4. 访问博客

访问 你的Github.github.io 查看你的 Hexo 博客

6 配置 GitHub 的Sshkey（可选）

1. 设置Git的user name和email

git config --global user.name "你的github用户名"
git config --global user.emali "你的邮箱"
输入cd ~/.ssh或返回主目录输入ls -a检查.ssh文件夹是否存在

2. 生成密钥

ssh-keygen -t rsa -C “你的邮箱
输入cd ~/.ssh目录内应包含 id_rsa ，id_rsa.pub 文件
添加密钥到ssh-agent
eval "$(ssh-agent -s)"
添加生成的SSH key到ssh-agent
ssh-add ~/.ssh/id_rsa

3. 添加密钥

登录Github，点击头像下的settings，点击SSH and GPG keys
点击New SSH key
输入随意的Title
Key中填写id_rsa.pub 文件 中的内容
点击Add SSH key添加密钥

4. 验证连接

ssh -T git@github.com
测试添加ssh是否成功。如果看到Hi后面是你的用户名，就说明成功了

问题：假如ssh-key配置失败，那么只要以下步骤就能完全解决首先，清除所有的key-pair
ssh-add -D
rm -r ~/.ssh
删除你在github中的public-key重新生成ssh密钥对
ssh-keygen -t rsa -C "xxx@xxx.com"
接下来正常操作
在github上添加公钥public-key:
1、首先在你的终端运行 xclip -sel c ~/.ssh/id_rsa.pub将公钥内容复制到剪切板
2、在github上添加公钥时，直接复制即可
3、保存
4、ssh -T git@github.com

7 创建第一篇博文

1. 创建博文

hexo new post “博文名”

2. 编辑博文

vi 博客目录/source/_posts/博文名
使用 Markdawn 来编辑你的博文吧
一般完整格式如下：

3. 生成及部署博文到 Github

hexo d -g

4. 访问 你的Github用户名.github.io 查看你的博客

9 切换主题

1. 主题选择

挑选你喜欢的主题，找到主题的github连接
Hexo主题

2. 主题下载

下载你的主题到本地博客目录的themes目录
git chone 主题github连接

3. 切换主题

修改本地博客目录_config.yml 中的
theme: landscape
改为
theme: 你的主题名称

4. 应用主题

hexo d -g
生成并上传github

5. 访问 你的Github用户名.github.io 查看你的博客
